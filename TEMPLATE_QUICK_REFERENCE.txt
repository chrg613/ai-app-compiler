================================================================================
API BINDING
================================================================================

TABLES:
  GET /api/user         → Loads table data
  DELETE /api/user/<id> → Delete row, refresh table

FORMS:
  POST /api/task        → Submit form to create record
  PUT /api/task/<id>    → Update existing record

All requests handled with:
  - Loading state ("Saving...")
  - Success/error messages
  - Auto-refresh of related tables

VERIFICATION FLOW
================================================================================

1. Generate an app through chat
2. Download ZIP file
3. Extract and install:
   $ unzip app_*.zip
   $ cd generated_app

   $ python app.py

   http://localhost:5000/ui/user

   - Table shows "Loading..."

   - Table refreshes automatically

   http://localhost:5000/api/user
